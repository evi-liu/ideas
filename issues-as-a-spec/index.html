Add expand/collapse all button for nested sub-issues #1079
Open
evi-liu opened 2 minutes ago

Problem

When viewing a project board with hierarchy enabled, users with deeply nested sub-issues (3+ levels) must click each parent row individually to expand the tree. With complex task decompositions, this becomes tedious.

Proposed Solution

Add an Expand All / Collapse All toggle button in the table header area of project board views that have hierarchy enabled.

Behavior

  • Expand All: Recursively expands every nested sub-issue level, eagerly fetching deeper levels
  • Collapse All: Collapses all rows back to top-level items only
  • State persists to session storage when persist flag is enabled

Acceptance Criteria

Expand/collapse all button visible in table header when hierarchy view is enabled
Expand all recursively fetches and displays all nested levels
Loading state shown during expand-all network requests
Keyboard accessible (treegrid pattern per #883)
Works correctly with sorting and filtering

Plan

Implementation plan for expand/collapse all

1 · Background & Problem

When viewing a project board with hierarchy enabled, users with deeply nested sub-issues (3+ levels) must click each parent row individually to expand the tree. With complex task decompositions (epic → feature → task → sub-task), this becomes tedious and makes it hard to get a full picture of the project.

2 · Objective

Add an Expand All / Collapse All toggle button in the table header area of project board views that have hierarchy enabled. The button eagerly fetches all nested levels, shows a loading indicator for large hierarchies, and persists state via the existing memex_sub_issues_persist_expanded_state flag.

3 · Tasks

Task 1 — Add ExpandCollapseAllButton component
Create a new React component in packages/memex/src/client/components/table/ that renders the expand/collapse toggle icon in the table header. Only visible when hierarchyEnabled is true. Complexity: Low.

Task 2 — Implement recursive expand/fetch logic
Extend useSubIssueExpansion hook to support expandAll() — recursively walk tree, trigger lazy fetches for unloaded children, update expansion state. Add collapseAll() to reset. Handle concurrent fetch coordination to avoid #1074. Complexity: High.

Task 3 — Add loading state for large hierarchy expansion
Show skeleton rows or spinner when expand-all triggers multiple network fetches. Debounce renders to avoid layout shift per #1060. Complexity: Medium.

Task 4 — Integrate with sorting, filtering, and persist state
Ensure expand-all respects active filters (only expand visible rows). Integrate with memex_sub_issues_persist_expanded_state. Handle #1071 sort-change flash. Complexity: Medium.

Task 5 — Keyboard accessibility & tests
Add keyboard shortcut per treegrid pattern (#883). Add Vitest/browser tests for expand-all, collapse-all, filters, and session storage persistence. Complexity: Medium.

4 · Related Issues

  • #883 — Keyboard expand/collapse (treegrid pattern)
  • #1071 — Polish for sort + persist state
  • #1074 — Fix over-requesting sub-issues
